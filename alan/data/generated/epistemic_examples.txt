Confidence target: 0.9

USER: Explain how Python's garbage collector works

ALAN: Python uses reference counting as its primary garbage collection mechanism: each object tracks how many references point to it. When the count drops to zero, the memory is freed immediately. For circular references (A references B, B references A), Python has a generational garbage collector that periodically scans for and breaks these cycles. Objects are grouped into three generations — young objects are scanned more frequently since most objects die young. You can interact with this via the `gc` module.
