--- EXAMPLE 3 ---
ID: reasoning_code_0002
Type: reasoning

Domain: code | Difficulty: medium

PROBLEM:
Debug this Python code that should find the second largest number in a list:
```python
def second_largest(nums):
    nums.sort()
    return nums[-2]
```
What's wrong and how to fix it?

THINKING:
  Step 1: Identify the bug — this fails when there are duplicate largest values
  Step 2: For [5, 5, 3], it returns 5 instead of 3
  Step 3: Also mutates the original list (side effect)
  Step 4: Fix: use a set to remove duplicates, then find second largest

SOLUTION:
The bug: if the list has duplicate maximum values like [5, 5, 3], `nums[-2]` returns 5 (the duplicate), not the actual second largest (3). Also, `.sort()` modifies the original list.

Fixed version:
```python
def second_largest(nums):
    unique = sorted(set(nums), reverse=True)
    if len(unique) < 2:
        return None  # Not enough unique values
    return unique[1]
```
The fix: convert to a set first to remove duplicates, then sort and take the second element.

VERIFICATION:
Test: [5, 5, 3] → unique sorted desc [5, 3] → returns 3 ✓. [1, 2, 3] → [3, 2, 1] → returns 2 ✓

--- EXAMPLE 5 ---
ID: reasoning_logic_0004
Type: reasoning

Domain: logic | Difficulty: medium

PROBLEM:
If all roses are flowers, and some flowers are red, can we conclude that some roses are red?

THINKING:
  Step 1: 'All roses are flowers' means roses are a subset of flowers
  Step 2: 'Some flowers are red' means there's an overlap between flowers and red things
  Step 3: But the red flowers might not include any roses — they could be tulips, poppies, etc.
  Step 4: We cannot logically conclude that any roses are red

SOLUTION:
No, we cannot conclude that some roses are red. The red flowers could all be non-rose flowers. This is a common logical fallacy — just because roses are flowers and some flowers are red, the 'red' subset might not overlap with the 'roses' subset.

VERIFICATION:
Counterexample: Roses={white roses}, Flowers={roses, red tulips}. All roses are flowers ✓. Some flowers are red (tulips) ✓. No roses are red ✓. Conclusion is invalid.

--- EXAMPLE 18 ---
ID: reasoning_science_0017
Type: reasoning

Domain: science | Difficulty: medium

PROBLEM:
Why does ice float on water, and why is this important for aquatic life?

THINKING:
  Step 1: Water expands when it freezes — ice is less dense than liquid water
  Step 2: This is unusual — most substances are denser as solids
  Step 3: The hydrogen bonds in ice form an open crystalline structure
  Step 4: Because ice floats, it insulates the water below, keeping it liquid
  Step 5: This allows fish and aquatic organisms to survive winter

SOLUTION:
Ice floats because water molecules form an open hexagonal crystal structure when freezing, making ice about 9% less dense than liquid water. This is critically important for aquatic life because the floating ice layer insulates the water below, preventing lakes and rivers from freezing solid. Without this property, aquatic ecosystems would be destroyed each winter.

VERIFICATION:
Density of ice ≈ 917 kg/m³, water ≈ 1000 kg/m³. Ice is less dense, so it floats ✓. Insulation effect prevents total freezing ✓.

--- EXAMPLE 20 ---
ID: reasoning_code_0019
Type: reasoning

Domain: code | Difficulty: hard

PROBLEM:
Implement a function to find all pairs of numbers in a list that sum to a target value. Optimize for time complexity.

THINKING:
  Step 1: Brute force is O(n²) — check every pair. We can do better.
  Step 2: Use a hash set. For each number, check if (target - number) is in the set.
  Step 3: This gives O(n) time complexity with O(n) space.
  Step 4: Handle edge cases: duplicate numbers, empty list, single element.
  Step 5: Return pairs without duplicates.

SOLUTION:
```python
def find_pairs(nums, target):
    seen = set()
    pairs = set()
    for num in nums:
        complement = target - num
        if complement in seen:
            pair = tuple(sorted((num, complement)))
            pairs.add(pair)
        seen.add(num)
    return list(pairs)

# Test
print(find_pairs([2, 4, 3, 5, 7, 8, 1], 9))  # [(2, 7), (4, 5), (1, 8)]
```
Time complexity: O(n). Space complexity: O(n). The key insight is using a hash set to check complements in O(1) instead of scanning the list.

VERIFICATION:
For target=9: 2→need 7(not seen), 4→need 5(not seen), 3→need 6(not seen), 5→need 4(seen!)→pair(4,5), 7→need 2(seen!)→pair(2,7), 8→need 1(not seen), 1→need 8(seen!)→pair(1,8) ✓

--- EXAMPLE 33 ---
ID: reasoning_code_0032
Type: reasoning

Domain: code | Difficulty: easy

PROBLEM:
Write a Python function to check if a string is a palindrome, ignoring spaces and case.

THINKING:
  Step 1: Define the function signature
  Step 2: Clean the string — remove spaces and convert to lowercase
  Step 3: Compare the cleaned string with its reverse
  Step 4: Return the boolean result

SOLUTION:
```python
def is_palindrome(s):
    cleaned = s.replace(' ', '').lower()
    return cleaned == cleaned[::-1]

# Test
print(is_palindrome('race car'))  # True
print(is_palindrome('hello'))     # False
```
The function strips spaces, lowercases, then checks if the string equals its reverse.

--- EXAMPLE 36 ---
ID: reasoning_code_0035
Type: reasoning

Domain: code | Difficulty: hard

PROBLEM:
Implement a function to find all pairs of numbers in a list that sum to a target value. Optimize for time complexity.

THINKING:
  Step 1: Brute force is O(n²) — check every pair. We can do better.
  Step 2: Use a hash set. For each number, check if (target - number) is in the set.
  Step 3: This gives O(n) time complexity with O(n) space.
  Step 4: Handle edge cases: duplicate numbers, empty list, single element.
  Step 5: Return pairs without duplicates.

SOLUTION:
```python
def find_pairs(nums, target):
    seen = set()
    pairs = set()
    for num in nums:
        complement = target - num
        if complement in seen:
            pair = tuple(sorted((num, complement)))
            pairs.add(pair)
        seen.add(num)
    return list(pairs)

# Test
print(find_pairs([2, 4, 3, 5, 7, 8, 1], 9))  # [(2, 7), (4, 5), (1, 8)]
```
Time complexity: O(n). Space complexity: O(n). The key insight is using a hash set to check complements in O(1) instead of scanning the list.

VERIFICATION:
For target=9: 2→need 7(not seen), 4→need 5(not seen), 3→need 6(not seen), 5→need 4(seen!)→pair(4,5), 7→need 2(seen!)→pair(2,7), 8→need 1(not seen), 1→need 8(seen!)→pair(1,8) ✓

--- EXAMPLE 44 ---
ID: reasoning_code_0043
Type: reasoning

Domain: code | Difficulty: medium

PROBLEM:
Debug this Python code that should find the second largest number in a list:
```python
def second_largest(nums):
    nums.sort()
    return nums[-2]
```
What's wrong and how to fix it?

THINKING:
  Step 1: Identify the bug — this fails when there are duplicate largest values
  Step 2: For [5, 5, 3], it returns 5 instead of 3
  Step 3: Also mutates the original list (side effect)
  Step 4: Fix: use a set to remove duplicates, then find second largest

SOLUTION:
The bug: if the list has duplicate maximum values like [5, 5, 3], `nums[-2]` returns 5 (the duplicate), not the actual second largest (3). Also, `.sort()` modifies the original list.

Fixed version:
```python
def second_largest(nums):
    unique = sorted(set(nums), reverse=True)
    if len(unique) < 2:
        return None  # Not enough unique values
    return unique[1]
```
The fix: convert to a set first to remove duplicates, then sort and take the second element.

VERIFICATION:
Test: [5, 5, 3] → unique sorted desc [5, 3] → returns 3 ✓. [1, 2, 3] → [3, 2, 1] → returns 2 ✓

--- EXAMPLE 66 ---
ID: reasoning_code_0065
Type: reasoning

Domain: code | Difficulty: hard

PROBLEM:
Implement a function to find all pairs of numbers in a list that sum to a target value. Optimize for time complexity.

THINKING:
  Step 1: Brute force is O(n²) — check every pair. We can do better.
  Step 2: Use a hash set. For each number, check if (target - number) is in the set.
  Step 3: This gives O(n) time complexity with O(n) space.
  Step 4: Handle edge cases: duplicate numbers, empty list, single element.
  Step 5: Return pairs without duplicates.

SOLUTION:
```python
def find_pairs(nums, target):
    seen = set()
    pairs = set()
    for num in nums:
        complement = target - num
        if complement in seen:
            pair = tuple(sorted((num, complement)))
            pairs.add(pair)
        seen.add(num)
    return list(pairs)

# Test
print(find_pairs([2, 4, 3, 5, 7, 8, 1], 9))  # [(2, 7), (4, 5), (1, 8)]
```
Time complexity: O(n). Space complexity: O(n). The key insight is using a hash set to check complements in O(1) instead of scanning the list.

VERIFICATION:
For target=9: 2→need 7(not seen), 4→need 5(not seen), 3→need 6(not seen), 5→need 4(seen!)→pair(4,5), 7→need 2(seen!)→pair(2,7), 8→need 1(not seen), 1→need 8(seen!)→pair(1,8) ✓

--- EXAMPLE 69 ---
ID: reasoning_code_0068
Type: reasoning

Domain: code | Difficulty: medium

PROBLEM:
Debug this Python code that should find the second largest number in a list:
```python
def second_largest(nums):
    nums.sort()
    return nums[-2]
```
What's wrong and how to fix it?

THINKING:
  Step 1: Identify the bug — this fails when there are duplicate largest values
  Step 2: For [5, 5, 3], it returns 5 instead of 3
  Step 3: Also mutates the original list (side effect)
  Step 4: Fix: use a set to remove duplicates, then find second largest

SOLUTION:
The bug: if the list has duplicate maximum values like [5, 5, 3], `nums[-2]` returns 5 (the duplicate), not the actual second largest (3). Also, `.sort()` modifies the original list.

Fixed version:
```python
def second_largest(nums):
    unique = sorted(set(nums), reverse=True)
    if len(unique) < 2:
        return None  # Not enough unique values
    return unique[1]
```
The fix: convert to a set first to remove duplicates, then sort and take the second element.

VERIFICATION:
Test: [5, 5, 3] → unique sorted desc [5, 3] → returns 3 ✓. [1, 2, 3] → [3, 2, 1] → returns 2 ✓

--- EXAMPLE 71 ---
ID: reasoning_code_0070
Type: reasoning

Domain: code | Difficulty: easy

PROBLEM:
Write a Python function to check if a string is a palindrome, ignoring spaces and case.

THINKING:
  Step 1: Define the function signature
  Step 2: Clean the string — remove spaces and convert to lowercase
  Step 3: Compare the cleaned string with its reverse
  Step 4: Return the boolean result

SOLUTION:
```python
def is_palindrome(s):
    cleaned = s.replace(' ', '').lower()
    return cleaned == cleaned[::-1]

# Test
print(is_palindrome('race car'))  # True
print(is_palindrome('hello'))     # False
```
The function strips spaces, lowercases, then checks if the string equals its reverse.

VERIFICATION:
Test: 'race car' → 'racecar' → reversed 'racecar' → True ✓. 'hello' → reversed 'olleh' → False ✓

```python
def is_palindrome(s):
    cleaned = s.replace(' ', '').lower()
    return cleaned == cleaned[::-1]

```
The function strips spaces, lowercases, then checks if the string equals its reverse.
