ID: image_screenshot_0010
Type: image_understanding

Image type: screenshot
Image: A screenshot of code with a bug in it

USER: Can you help me find the bug in this code snippet?

ALAN: The code you've shared correctly implements the factorial function for non-negative integers, but it lacks validation for negative inputs. If you call factorial with a negative number, it will recurse indefinitely and eventually cause a RecursionError. To fix this, you should add a check at the start of the function to raise an exception or return an error when n is negative. For example:

```python
def factorial(n):
    if n < 0:
        raise ValueError('Input must be a non-negative integer')
    if n == 0:
        return 1
    return n * factorial(n - 1)
```

This will prevent infinite recursion and make your function more robust.
